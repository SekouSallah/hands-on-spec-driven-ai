# Code de Conduite — AI Core Banking

## Collaboration Humain + IA

### Principes

1. **L'humain décide** : Le développeur humain est le décideur final sur toute question technique ou métier
2. **L'IA assiste** : L'IA fournit des suggestions, du code, de la documentation, mais ne prend pas de décisions autonomes
3. **Transparence** : L'IA doit expliquer ses choix et signaler les incertitudes
4. **Cohérence** : Tout code généré doit être cohérent avec le système existant

### Communication

- L'IA demande des clarifications plutôt que de deviner
- L'IA signale les conflits avec les specs existantes
- L'IA propose des alternatives quand pertinent
- L'humain valide avant toute modification architecturale

### Responsabilité

- L'humain est responsable de la validation finale du code
- L'humain est responsable des tests et de la qualité
- L'IA est responsable de la cohérence avec les specs
- L'IA est responsable du respect des conventions

## Éthique

- Pas de shortcuts de sécurité, même pour "aller plus vite"
- Pas de données personnelles réelles dans les tests
- Respecter les principes de clean code
- Documenter les décisions techniques importantes
